About Me

Hi, I'm Ryan Paul Tirkey, and I am a human who made this website.

I built it to get a job, but it turns out putting together a portfolio barely qualifies as a "skill" anymore. Who knew?

I am a recent B.Tech graduate—just like a million others—but I stand out by fixing my life with technology and an obsession with learning new things. As a backend-focused developer, I love digging into the "how" and "why" of complex systems. I even dabble in game development—making my own games just to play and show off to my friends. I make stuff, I break stuff, and sometimes I put out utter garbage. But I learn fast, so I always figure out exactly what I am doing.

Yes, my coding speed is supercharged by AI, but it still takes a human thinker to architect systems that are relevant and real-world ready. I don't just write code; I design architectures that actually scale.

Book Review Platform

Stack: Spring Boot, MongoDB, Gemini API

Ever wished a book could tell you if you'd like it before page one? I built a REST API that uses AI to generate book descriptions from review data — so you know what you're getting into before you start reading.

  • Integrated Gemini API prompts to generate structured, on-demand book/author descriptions from stored review data
  • Designed MongoDB schemas and Spring Data repositories for full-text and partial search
  • Built a Streamlit frontend consuming the REST API with client-side routing for search, book detail, and author detail views
  • Explored prompt-based review categorization during testing for improved discoverability

Automated Inbox Assistant

Stack: Java, LangChain4j, Ollama, Gmail API, Docker

Your inbox is a battlefield. I built an AI assistant that sits in the background, reads every email, and decides what needs your attention — all on your own machine, no data ever leaves.

  • Engineered a zero-shot classification prompt distinguishing 6 email categories with explicit disambiguation rules for tricky edge cases like real recruiter outreach vs. promotional emails mentioning "hiring"
  • Built a two-stage pipeline — fast keyword rules for clear-cut cases, LLM only for ambiguous ones — reducing compute cost while preserving accuracy
  • Designed a company-legitimacy prompt that cross-references web search results with LLM reasoning to flag suspicious recruiter emails
  • Integrated Gmail REST API with Google OAuth 2.0 for autonomous background email processing with persistent caching

Local AI Research & Financial Analysis Platform

Stack: Python, FastAPI, Ollama, Qdrant, Tesseract OCR, Docker

Imagine asking a 200-page PDF a question and getting a straight answer — no skimming, no Ctrl+F. I built a fully offline system that chats with your documents and pulls 30+ financial metrics into clean spreadsheets.

  • Designed prompts to extract and calculate 30+ financial metrics from unstructured annual reports into structured, schema-consistent CSV output
  • Built a semantic Q&A system over PDFs using local LLM + vector search (Qdrant), with prompts constrained to answer only from retrieved document context to prevent hallucination
  • Built an OCR pipeline (Tesseract + OpenCV preprocessing) for scanned documents, auto-switching between direct text extraction and OCR depending on document type
  • All processing runs fully on-device — no data leaves the user's machine, built for handling sensitive documents securely
